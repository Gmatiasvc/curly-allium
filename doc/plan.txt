Travel+ (Curly allium)


Funcionamiento propuesto backend

1. Handshake cliente-servidor

2. Establecimiento de una comunicación segura utilizando ML-KEM o X25519 y Ed25519, depende de lo que sea más fácil implementar.

3. Login según la request enviada. Código 100 para clientes, código 101 para administrador.

4. Enviar y recibir requests al servidor utilizando los códigos considerados debajo.


Consideraciones para backend

 - Usuarios: Lista de amigos usando listas simples, estas listas se pueden utilizar para generar grafos tomándolas con listas de adyacencia.
 - Paraderos: Grafo no dirigido con pesos en las aristas, considero conveniente utilizar lista de adyacencia.
 - Base de datos: Mantener los datos de los clientes encriptados. Somos mejores que la reniec. Decidan si usar o no MySQL o SQLite3 o PostgreSQL, se puede hacer mucho con Postgre. Al menos las passwords hasheadas con salt.
